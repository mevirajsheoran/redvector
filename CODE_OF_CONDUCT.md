# Code of Conduct

## Our Standard

ThreatForge contributors are expected to be respectful, constructive, and safety-minded.

Examples of positive behavior:

- Giving actionable and kind feedback
- Welcoming contributors with different experience levels
- Prioritizing education, clarity, and responsible security practice
- Respecting project boundaries around authorized use only

Examples of unacceptable behavior:

- Harassment, intimidation, or discriminatory language
- Publishing unsafe guidance for unauthorized use
- Dismissing safety concerns without discussion
- Trolling, personal attacks, or deliberately disruptive behavior

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, or contributions that violate this code of conduct.

## Reporting

If you experience or witness unacceptable behavior, report it through GitHub to the maintainer. Sensitive reports should be handled privately when possible.
