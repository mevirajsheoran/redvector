# Security Policy

## Supported Scope

ThreatForge is an educational offensive security framework intended for authorized lab use. Security reports are especially helpful for:

- Target validation bypasses
- Unsafe defaults
- Privilege or sandbox escape issues in helper scripts
- Secrets accidentally committed to the repository
- Dashboard or API vulnerabilities that affect local users

## Reporting a Vulnerability

Please do not open a public issue for a sensitive vulnerability.

Instead:

1. Open a private GitHub security advisory if available for this repository.
2. If that is not available, contact the maintainer through GitHub and clearly label the report `Security: ThreatForge`.
3. Include reproduction steps, impact, and any suggested mitigation.

## Response Expectations

Best effort targets:

- Initial acknowledgement within 7 days
- Triage and severity assessment as soon as practical
- A fix or mitigation plan for confirmed issues when feasible

## Safe Disclosure

Please avoid publishing proof-of-concept details until the issue has been reviewed and maintainers have had a reasonable opportunity to respond.
